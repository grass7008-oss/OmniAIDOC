# 知識標籤 Knowledge Tags

### 模組說明：知識標籤 (Knowledge Tags)

{% hint style="success" %}
功能定義： 本功能用於建立與管理「知識標籤庫」。標籤可用於對知識內容進行關鍵字標記，打破傳統分類的限制，提升搜尋的精準度與資料關聯性
{% endhint %}

#### 1. 介面欄位說明

* 標籤名稱： 標籤的顯示文字（如：食品安全A、藥妝類別A）。
* 編號： 系統唯一代碼（如 `KT000000001`），用於資料庫關聯。
* 創立者/修改人： 紀錄建立與最後維護該標籤的帳號，便於權限追蹤。
* 創立時間/修改時間： 標籤的完整生命週期紀錄。
* 是否刪除： 顯示該標籤目前的啟用狀態。

#### 2. 標準操作流程

**A. 新增標籤**

1. 點擊列表左上方藍色 \[+ 新增] 按鈕。
2. 在彈窗中輸入標籤名稱，建議以簡短、具代表性的關鍵字為主。
3. 儲存後，該標籤即可在編輯「知識內容」時被選用。

**B. 批次與匯出管理**

1. 批次刪除： 若需清理無效標籤，可勾選多筆紀錄後點擊紅色的 \[刪除] 按鈕。
2. 資料匯出： 點擊藍色 \[匯出] 按鈕，可將現有的標籤清單匯出，方便跨系統比對或進行標籤一致性檢查。

**C. 檢索標籤**

1. 使用右側搜尋框，可透過「標籤名稱」進行模糊搜尋。
2. 點擊 \[展開] 可顯示更多進階篩選條件（如依創立時間、創立者篩選）。

***

{% hint style="warning" %}


#### <mark style="color:$primary;">管理建議 (Best Practices)</mark>

* <mark style="color:$primary;">避免重複： 建立標籤前建議先進行搜尋，避免產生「食安」與「食品安全」這類含義重複但文字不同的標籤，維持資料乾淨度。</mark>
* <mark style="color:$primary;">多維度應用： 一則知識內容可以同時掛載多個標籤。例如一筆法規文件可以同時掛上「食安科」與「急件」兩個標籤。</mark>
{% endhint %}
